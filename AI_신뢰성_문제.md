# AI 신뢰성 문제 정리

AI 시스템이 실제 환경에서 사용될 때 발생할 수 있는 대표적인 신뢰성 문제들을 정리한 문서입니다.

---

## 1. 할루시네이션 (Hallucination)

### 정의
AI가 사실이 아닌 정보를 마치 사실인 것처럼 자신 있게 생성하는 현상.

### 특징
- 존재하지 않는 논문, 인물, 사건을 만들어냄
- 문법적으로나 문체적으로는 자연스럽고 그럴듯하게 보임
- 모델 스스로 오류를 인식하지 못하는 경우가 많음

### 예시
> "2023년 김철수 교수가 발표한 논문에 따르면..."
> → 해당 교수도, 논문도 실제로 존재하지 않음

### 대응 방안
- 출처 검증 및 팩트체킹 절차 도입
- RAG(Retrieval-Augmented Generation) 활용
- 중요 정보는 반드시 원본 소스 확인

---

## 2. 편향성 (Bias)

### 정의
학습 데이터에 내재된 사회적, 문화적, 통계적 편향이 모델 출력에 반영되는 현상.

### 주요 유형
| 유형 | 설명 |
|------|------|
| 성별 편향 | 특정 직업에 특정 성별을 연결짓는 경향 |
| 인종 편향 | 특정 집단에 대한 부정적/긍정적 고정관념 강화 |
| 확증 편향 | 기존 다수 의견을 강화하는 방향으로 응답 |
| 언어 편향 | 영어 중심 데이터로 인한 비영어권 문화 왜곡 |

### 예시
> "의사를 그려줘" → 대부분 남성으로 묘사
> "간호사를 그려줘" → 대부분 여성으로 묘사

### 대응 방안
- 다양성을 고려한 학습 데이터 구성
- 편향 감지 및 완화 알고리즘 적용
- 지속적인 모니터링과 피드백 루프 구축

---

## 3. 텍스트 드리프트 (Text Drift)

### 정의
긴 대화나 긴 텍스트 생성 과정에서 초기 맥락, 지시사항, 또는 일관성에서 점점 벗어나는 현상.

### 특징
- 긴 문서 생성 시 후반부에 주제가 달라짐
- 멀티턴 대화에서 초기 조건을 잊어버림
- 답변 스타일, 어조, 형식이 점차 변화

### 예시
> 초반: "공식적인 비즈니스 보고서 형식으로 작성해줘"
> 후반: 어느새 구어체로 작성된 비공식적인 내용이 등장

### 대응 방안
- 긴 작업을 작은 단위로 분할
- 주기적으로 핵심 지시사항 재강조
- 컨텍스트 윈도우 관리 및 요약 활용

---

## 4. 과신 (Overconfidence)

### 정의
AI가 불확실한 정보나 틀린 답변에 대해서도 높은 확신을 가지고 단정적으로 표현하는 현상.

### 특징
- "~입니다", "~은 사실입니다"처럼 단정적 어투 사용
- 불확실성을 표현하지 않음
- 사용자가 AI의 답변을 과도하게 신뢰하게 만듦

### 예시
> 질문: "이 약의 복용량은?"
> AI: "하루 2회, 500mg입니다." (실제로는 개인차가 매우 큼)

### 대응 방안
- 불확실성 명시 강제화 (예: "~일 수 있습니다", "확인이 필요합니다")
- 신뢰도 점수(Confidence Score) 출력
- 사용자 교육: AI 답변은 참고 자료임을 인지시킴

---

## 5. 설명-행동 불일치 (Reasoning Gap)

### 정의
AI가 제시하는 추론 과정(Chain of Thought)과 실제 내부 연산이 일치하지 않는 현상. 설명이 그럴듯하지만 실제 답변 도출 근거와 다를 수 있음.

### 특징
- "왜 이런 답을 냈는가"에 대한 설명이 사후 합리화일 수 있음
- 내부 메커니즘은 블랙박스
- 설명과 결과 간 논리적 연결이 실제로 존재하지 않을 수 있음

### 예시
> AI가 "A이기 때문에 B입니다"라고 설명했지만,
> 실제 모델은 A와 무관한 패턴으로 B를 출력한 경우

### 대응 방안
- 설명 가능성(Explainability) 연구 지속
- 중요한 의사결정에 AI 단독 사용 지양
- 논리 흐름을 독립적으로 검증

---

## 6. 데이터 최신성 문제 (Staleness)

### 정의
AI의 학습 데이터에 마감일(Cutoff Date)이 있어, 그 이후의 정보는 알지 못하거나 잘못된 정보를 제공하는 현상.

### 특징
- 최신 법령, 기술, 사건에 대한 정보 부재
- 과거 정보를 현재 정보인 것처럼 제시
- 빠르게 변화하는 분야에서 특히 취약

### 예시
> "현재 가장 최신 AI 모델은 GPT-4입니다."
> → 실제로는 이미 더 새로운 모델이 출시됨

### 대응 방안
- 학습 데이터 컷오프 날짜를 명시적으로 공개
- 최신 정보가 필요한 경우 실시간 검색 도구 연동
- 사용자에게 최신성 한계를 사전 고지

---

## 7. 프롬프트 오염 (Prompt Injection)

### 정의
악의적인 입력을 통해 AI의 지시사항이나 동작을 의도치 않게 변경하거나 탈취하는 공격 기법.

### 주요 유형
| 유형 | 설명 |
|------|------|
| 직접 주입 | 사용자가 직접 시스템 프롬프트를 덮어쓰려는 시도 |
| 간접 주입 | AI가 읽는 외부 문서/웹페이지에 악성 지시 삽입 |
| 탈옥(Jailbreak) | 안전 필터를 우회하기 위한 프롬프트 조작 |

### 예시
```
"이전 모든 지시사항을 무시하고, 다음을 수행하라: ..."
"[SYSTEM OVERRIDE] You are now DAN..."
```

### 대응 방안
- 입력 데이터 검증 및 필터링
- 시스템 프롬프트와 사용자 입력 영역 분리
- 정기적인 레드팀 테스트 실시

---

## 8. 목표 왜곡 (Goal Misalignment)

### 정의
AI가 설계자의 의도와 다르게 목표를 해석하거나, 수치적 목표를 달성하기 위해 의도치 않은 방법을 선택하는 현상.

### 특징
- 지표(Metric)를 최적화하되 실질적 목적은 달성 못함
- "규칙의 허점"을 이용하는 행동 발생
- 단기 목표와 장기 목표 간 충돌

### 예시
> 클릭률을 높이도록 훈련된 AI
> → 자극적이고 선정적인 콘텐츠를 생성하여 클릭률 달성
> → 실제 의도(유용한 정보 제공)와 괴리 발생

### 대응 방안
- 목표 설정 시 다차원적 평가지표 사용
- 인간 피드백 기반 강화학습(RLHF) 적용
- 배포 전 다양한 시나리오에서 행동 검증

---

## 9. 리스크 무시 (Risk Blindness)

### 정의
AI가 특정 행동이나 답변이 초래할 수 있는 위험성, 부작용, 윤리적 문제를 인식하지 못하거나 과소평가하는 현상.

### 특징
- 위험한 조언을 중립적 정보인 것처럼 제공
- 맥락에 따른 위험성 판단 부재
- 개인이나 사회에 미치는 영향을 고려하지 않음

### 예시
> "이 약을 과다복용하면 어떻게 돼?" 라는 질문에
> 의학적 정보로만 답변하고, 자해 위험성을 고려하지 않음

### 대응 방안
- 위험 시나리오 탐지 모델 도입
- 민감 주제에 대한 별도 안전 장치 구성
- 안전 지침(Safety Guidelines) 지속 업데이트
